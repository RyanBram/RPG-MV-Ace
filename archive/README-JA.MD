# RPG Maker MV コアスクリプト アーカイブ

## 概要

このアーカイブには、複数のバージョンの Pixi.js レンダリングライブラリにわたって段階的にポートされた RPG Maker MV コアスクリプトの集合が含まれています。このアーカイブは、新しい Pixi バージョンを活用するために更新されてきたコアスクリプトの進化と互換性を追跡しています。

## Pixi バージョン対応

このアーカイブには、以下の Pixi バージョンにポートされたビルド済みコアスクリプトバージョンが含まれています：

-   **Pixi v4** — RPG Maker MV の元のリリース
-   **Pixi v5** — パフォーマンスと安定性の改善
-   **Pixi v7** — 最新のウェブレンダリング標準（Ace / v2.0）
-   **Pixi v8** — WebGPU と次世代レンダリング

_注：Pixi v6 はこのアーカイブには含まれていません。_

## アーキテクチャ & プラグイン互換性

RPG Maker MV コアスクリプトの重要な強みは、**小島勇次**によって設計された堅牢な抽象化レイヤーです。この優れた抽象化により、以下が保証されます：

1. **変更の分離**: Pixi バージョンアップグレードは `rpg_core.js` モジュールに分離されており、このモジュールはゲームエンジンとレンダリングライブラリの間の抽象化レイヤーとして機能します。
2. **プラグイン互換性**: プラグインエコシステムは大規模な破壊的変更から保護されています。RPG Maker MV 用に書かれたプラグインは、Pixi 内部に直接依存しない限り、修正なしに異なる Pixi バージョン間で機能し続けることができます。
3. **前方互換性**: 新しいバージョンのコアスクリプトは最新の Pixi 機能を採用できますが、ゲームロジック、マネージャー、シーン、およびスプライトの公開 API は同じに保たれます。

この設計哲学により、Pixi v4 から v5 を経由して v7 への円滑な移行が可能になり、一方、何千ものコミュニティプラグインの完全性が保持されています。

## パフォーマンス期待値

**重要な注意**: より新しい Pixi バージョンへのアップグレードは、自動的にはゲームパフォーマンスの向上をもたらしません。むしろ：

-   Pixi バージョンアップグレードは、レンダリング効率、WebGL 安定性、およびブラウザ互換性に対する**基盤的な改善**を提供します。
-   実際のパフォーマンス向上を実現するには、コアスクリプトのアーキテクチャを調整し、Pixi の機能を完全に活用する必要があります。たとえば、テクスチャバッチ処理、レンダリング状態管理の最適化、または新しいレンダリング機能の利用です。
-   適切に調整された RPG Maker MV ゲームが Pixi v7 で実行される場合、Pixi v5 での実行よりもパフォーマンスが優れていますが、これには単なるライブラリスワップ以上の意図的な最適化作業が必要です。

開発者は以下を推奨されます：

-   ゲームをプロファイリングしてボトルネックを特定する。
-   複数のデバイスと Pixi バージョン全体でテストする。
-   各 Pixi バージョンの強みを活かすために、ターゲットされた最適化を適用する。

## このアーカイブの使用方法

このアーカイブの各コアスクリプトバージョンは**ビルド済みで、すぐに使用できます**。コンパイルは不要です。特定のバージョンを統合するには：

1. 目的の Pixi バージョンフォルダーからコアスクリプトファイルを抽出します。
2. RPG Maker MV プロジェクトの `js/` ディレクトリに直接コピーします。
3. ステージング環境でプロジェクトとプラグイン付きのコアスクリプトをテストします。
4. パフォーマンス指標を監視して Pixi バージョンの影響を評価します。
